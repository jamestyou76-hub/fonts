# Word 报告样式模板：A4 竖向

适用场景：正式报告、客户要求整理书、设计概要、会议记录、简要说明文档等以文字说明为主的技术文档。

## 1. 页面设定

| 项目 | 设定 |
|---|---|
| 纸张 | A4 |
| 方向 | 竖向 Portrait |
| 页面尺寸 | 21.0 cm × 29.7 cm |
| 页边距 | 上 15 mm / 下 15 mm / 左 16 mm / 右 16 mm |
| 页眉页脚 | 页眉 8 mm / 页脚 8 mm |
| 推荐用途 | 正式报告、客户要求整理、设计概要、少列数表格 |

## 2. 字体与段落

| 项目 | 设定 |
|---|---|
| 主字体 | 等线 / DengXian |
| 英数字字体 | 等线 / DengXian |
| 日文文档 fallback | 等线优先；若日文假名显示异常，可用 Yu Gothic / Meiryo 作为 fallback |
| 正文字号 | 10 pt |
| 表格字号 | 9 pt |
| 标题 | 16～18 pt，加粗 |
| 一级标题 | 13 pt，加粗，蓝绿色或深灰色 |
| 二级标题 | 11～12 pt，加粗 |
| 段前/段后 | 标题段后 4～6 pt；正文段后 3 pt |
| 行距 | 单倍或固定值 13～14 pt |

## 3. 颜色规范

| 用途 | 颜色 | Hex |
|---|---|---|
| 表头底色 | 蓝绿色 | `#009999` |
| 表头文字 | 白色 | `#FFFFFF` |
| 奇数数据行 | 白色 | `#FFFFFF` |
| 偶数数据行 | 浅青绿色 | `#EAF7F7` |
| 表格边框 | 黑色 | `#000000` |
| 一级标题辅助色 | 深蓝绿色 | `#006666` |
| 注意/未确定 | 深橙色，谨慎使用 | `#C65911` |

## 4. 表格风格

| 项目 | 设定 |
|---|---|
| 表格边框 | 全部黑色实线 |
| 边框线宽 | 0.75 pt，必要时外框 1.0 pt |
| 表头 | 蓝绿色底色，白色文字，加粗，居中 |
| 数据行 | 隔行底色：白色 / 浅青绿色 |
| 单元格内边距 | 上下 1.5～2 mm，左右 1.5～2 mm |
| 对齐 | 项目/状态列居中；说明列左对齐 |
| 换行 | 自动换行，避免单元格过宽 |
| 表格宽度 | 100% 页面可用宽度 |
| 图片说明 | “图 1：xxx” 格式，置于图片下方 |

## 6. 内容写法规则

- 正式报告建议先给“结论/概要”，再给详细表格。
- 每个章节尽量控制在半页以内，方便客户快速阅读。
- 未确定项目统一写为 `TBD`。
- 表格内容使用短句，长说明放在正文段落中。
- 若文档只用于“设计需求”，不要包含测试、验证、实测、后续措施等内容。
- 图表下方必须加简短说明，避免图片单独出现。

## 7. Word 生成实现要点

使用 `python-docx` 生成 Word 时建议：

- `section.orientation = WD_ORIENT.PORTRAIT`
- `section.page_width = Cm(21.0)`
- `section.page_height = Cm(29.7)`
- 所有 paragraph/run 设置 East Asian font 为 `等线`
- 表头填充 `009999`
- 表头文字设为白色、加粗
- 数据行隔行填充 `FFFFFF` / `EAF7F7`
- 所有单元格边框设置为黑色 `000000`
- 图片宽度建议不超过页面可用宽度的 90～100%

## 8. 文件命名建议

`项目名_文档主题_A4竖向_语言_日期.docx`

示例：

`バッテリー充電基板_顧客要求整理書_A4縦_JP_20260620.docx`
