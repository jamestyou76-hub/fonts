# ODS 中日文 Word 报告样式：Noto Sans CJK

适用场景：面向 `有限会社ODS` 的中文、日文或中日文混排技术报告、设计审查报告、客户要求整理书、工程 TODO、器件评估和验证说明。

本样式参考 `工程技术报告_Word风格规范.md`、`Word报告样式模板_A4竖向.md`、`Word报告样式模板_A4横向.md` 和 `PDF_样式_NotoSansCJK_复用说明.md`。页面横向或竖向按内容自行选择，本规范不强制指定方向。

## 1. 文档定位

- 文档类型：正式工程技术报告、设计审查报告、设计需求整理、工程 TODO、评估 memo。
- 设计原则：专业、克制、信息密度适中、结论优先、便于客户快速扫描。
- 内容顺序：先给结论和风险，再给依据、表格、计算、图示和验证项。
- 语言原则：中文、日文、英文、数字、器件型号、网络名和页眉页脚统一排版，不混用系统默认宋体、MS Gothic 或 SimSun。
- 未确定事项统一写为 `TBD`，不要写成确定结论。

## 2. 页面设定

| 项目 | 设定 |
|---|---|
| 纸张 | A4 |
| 方向 | 按内容选择，文字报告优先竖向，宽表格优先横向 |
| 页边距 | 竖向建议：上 18-19 mm / 下 18 mm / 左右 20-21.5 mm；横向建议：上下左右 12-15 mm |
| 页眉页脚 | 页眉 8-9 mm / 页脚 8-9 mm |
| 正文有效宽度 | 表格、图片和提示框不得超出页面可用宽度 |
| 推荐用途 | 中日文技术报告、客户提交资料、设计需求表、风险清单、工程 TODO |

## 3. 字体与链接

中文和日文统一使用 Noto Sans CJK。日文文档或中日文混排优先使用 `Noto Sans CJK JP`；简体中文为主且日文较少时可使用 `Noto Sans CJK SC`。

只列 TTF 文件，Word 使用场景下 TTF 已足够。若后续用于 LaTeX/PDF，也可按工具支持情况改用同目录 OTF。

| 用途 | 字体 | GitHub 文件链接 |
|---|---|---|
| 中日文正文，优先推荐 | Noto Sans CJK JP Regular | https://github.com/wheregone/fonts/blob/main/doc-font/NotoSansCJKjp-Regular.ttf |
| 中日文粗体，优先推荐 | Noto Sans CJK JP Bold | https://github.com/wheregone/fonts/blob/main/doc-font/NotoSansCJKjp-Bold.ttf |
| 简体中文正文备用 | Noto Sans CJK SC Regular | https://github.com/wheregone/fonts/blob/main/doc-font/NotoSansCJKsc-Regular.ttf |
| 简体中文粗体备用 | Noto Sans CJK SC Bold | https://github.com/wheregone/fonts/blob/main/doc-font/NotoSansCJKsc-Bold.ttf |

Word 字体属性要求：

- 中文、日文、英文、数字、器件型号、网络名、表格、图题、页眉页脚均设置为同一字体族。
- Word 的西文、复杂文字和东亚文字字体属性都设置为 `Noto Sans CJK JP` 或 `Noto Sans CJK SC`。
- 同一份文档内不要混用 `Noto Sans CJK JP` 和 `Noto Sans CJK SC` 作为主字体；除非确有字形需求，只保留另一套作为 fallback。
- 若必须交付可稳定复现的 Word，建议嵌入字体或同时提供上述 TTF。

## 4. 字号与层级

| 元素 | 字号 | 字重 | 颜色 | 段落间距 |
|---|---:|---|---|---|
| 报告主标题 | 22-25 pt | 粗体 | 深蓝绿 `#084C4E` | 段后 8 pt |
| 一级标题 | 15-16 pt | 粗体 | 深蓝绿 `#084C4E` | 段前 14-15 pt，段后 6-7 pt |
| 二级标题 | 12-12.5 pt | 粗体 | 蓝绿色 `#0F6B6D` | 段前 10-11 pt，段后 5 pt |
| 三级标题 | 10.5-10.8 pt | 粗体 | 深灰 `#1F2933` | 段前 7-8 pt，段后 3 pt |
| 正文 | 9.5-10 pt | 常规 | 深灰 `#1F2933` | 段后 4-5 pt，1.12-1.18 倍行距 |
| 表格正文 | 8.2-9.5 pt | 常规 | 深灰 `#1F2933` | 单元格内紧凑行距 |
| 图题/注释 | 8.5-9 pt | 常规 | 灰色 `#5B6573` | 居中或左对齐 |
| 页眉页脚 | 8.2-8.5 pt | 常规 | 灰色 `#5B6573` | 紧凑 |

要求：

- 标题使用 Word 标题样式，不手工模拟。
- 标题与其后正文保持在同一页，避免标题孤立在页尾。
- 正文使用短段落；长说明优先拆成条目、表格或提示框。
- 英文器件型号、网络名和计算符号保持原文，不强制翻译。

## 5. 配色

| 用途 | 色值 |
|---|---|
| 主色 / 表头 | `#0F6B6D` |
| 深色标题 | `#084C4E` |
| 正文文字 | `#1F2933` |
| 次要文字 | `#5B6573` |
| 表格隔行底色 | `#EAF3F3` |
| 浅灰辅助底色 | `#F3F6F7` |
| 严重风险 | `#A61B1B` |
| 警告与待确认 | `#8A5A00` |
| 通过与正向结论 | `#176B45` |
| 表格边框 | 黑色 `#000000` |

正文不大面积使用彩色。颜色只用于标题、表头、风险等级、结论标签和少量强调。

## 6. 首页结构

首页推荐采用工程报告式信息结构：

1. 文档类别短标签，例如 `設計レビュー報告書`、`技術分析報告書`、`设计需求整理书`。
2. 报告主标题。
3. 审查对象、文件版本、资料基线或项目名。
4. 总体结论提示框。
5. 基本信息表：项目、对象、核心器件、范围、日期、版本。
6. 关键结论速览或风险清单。

不要制作只有大标题和大量留白的装饰性封面。

## 7. 表格规范

- 表格用于参数对比、状态矩阵、风险清单、客户要求整理和验证矩阵。
- 表格宽度与页面可用宽度一致，必要时明确固定列宽。
- 表头使用蓝绿色 `#0F6B6D`，白色粗体文字。
- 正文行使用白色和浅蓝绿色 `#EAF3F3` 隔行显示。
- 所有边框使用黑色实线，线宽 0.75 pt，必要时外框 1.0 pt。
- 单元格垂直居中；编号、状态、优先级、数值列居中；说明列左对齐。
- 单元格内边距建议上下 1.5-2 mm，左右 1.5-2 mm。
- 不设置固定行高，允许文字自动换行。
- 跨页表格必须重复表头。
- 风险等级列可用红色或橙色文字突出，但避免整行高饱和底色。

推荐列结构：

| 表格类型 | 推荐列 |
|---|---|
| 客户要求整理 | No. / 要求項目 / 要求内容 / 対応方針 / 備考 |
| 工程 TODO | No. / 分類 / 現状 / TODO / 優先度 / TBD |
| 技术风险表 | No. / リスク / 影響 / 対応 / クローズ条件 |
| 参数核查表 | 項目 / 現在値 / 要求値 / 判定 / 根拠 |

## 8. 提示框规范

提示框使用单列表格实现，不使用装饰性阴影或大圆角。

| 类型 | 边框/标签颜色 | 底色 | 典型标签 |
|---|---|---|---|
| 严重风险 | `#A61B1B` | `#FCECEC` | 必須修正 / 必须修正 |
| 警告 | `#8A5A00` | `#FFF6DD` | 要確認 / 需确认 |
| 正向结论 | `#176B45` | `#EAF6EF` | 判定OK / 已通过 |
| 普通结论 | `#084C4E` | `#EAF3F3` | 総合結論 / 总体结论 |

每个提示框只表达一个核心判断。标签使用粗体和彩色文字，说明正文仍使用深灰色。

## 9. 图片与图题

- 只截取与当前分析直接相关的电路、PCB、测试波形或表格区域。
- 图片宽度通常为页面可用宽度的 90%-100%。
- 保持原始比例，不拉伸变形。
- 图题置于图片下方，使用灰色 8.5-9 pt 字体。
- 图题格式可用 `図 N  説明（出典：xxx）` 或 `图 N  说明（来源：xxx）`，全文保持一致。
- 图片前后保留适度间距，不让图片贴住标题、表格或页脚。

## 10. 页眉与页脚

页眉必须加入日文公司名：

```text
有限会社ODS
```

推荐页眉左侧显示：

```text
有限会社ODS | 项目名称 | 文档类型
```

日文文档可写为：

```text
有限会社ODS | プロジェクト名 | 文書種別
```

页脚左侧建议显示：

```text
资料日期：YYYY-MM-DD | 报告版本：Rev.X.X
```

日文文档可写为：

```text
資料日付：YYYY-MM-DD | 報告版数：Rev.X.X
```

页脚右侧显示自动页码：

```text
第 X 页
```

或日文：

```text
X / Y ページ
```

页眉页脚使用 `Noto Sans CJK JP` 或 `Noto Sans CJK SC`，字号 8.2-8.5 pt，颜色 `#5B6573`，不添加粗重装饰线。

## 11. Word 生成实现要点

使用 `python-docx` 或其他工具生成 Word 时建议：

- 不强制设置横向或竖向；根据内容宽度选择页面方向。
- 全文所有 run 设置 `w:eastAsia`、`w:ascii`、`w:hAnsi` 和 `w:cs` 字体为同一 Noto Sans CJK 字体。
- 日文和中日文混排默认 `Noto Sans CJK JP`；简体中文主文档可用 `Noto Sans CJK SC`。
- 页眉创建后写入 `有限会社ODS | 项目名称 | 文档类型`。
- 表头填充 `0F6B6D`，表头文字白色、加粗。
- 数据行隔行填充 `FFFFFF` / `EAF3F3`。
- 所有单元格边框设置为黑色 `000000`。
- 长表格允许跨页，并设置表头重复。
- 不使用空格或制表符模拟对齐，不使用固定行高截断文字。
- 文档元数据至少填写标题、主题、关键词和作者/公司信息。

## 12. 交付前验收

生成 Word 后必须检查：

- 渲染为 PDF 或 PNG，逐页确认没有缺字、方框、文字重叠或裁切。
- 确认中文、日文、英文、数字、页眉页脚和表格字体均为 Noto Sans CJK。
- 确认页眉含有 `有限会社ODS`。
- 确认表格没有超出页边距，长文本能够正常换行。
- 确认跨页表格重复表头。
- 确认图片清晰、比例正确、图题与图片保持在一起。
- 确认标题没有孤立在页尾。
- 确认页码、页眉、页脚和版本信息一致。

## 13. 后续使用指令

后续需要生成同风格 Word 时，可直接使用：

> 按《ODS 中日文 Word 报告样式：Noto Sans CJK》生成 Word。页面方向按内容决定，不必特别指定横竖；中文、日文、英文、数字、器件型号、表格、图题及页眉页脚统一使用 Noto Sans CJK JP，必要时简体中文文档可用 Noto Sans CJK SC；页眉加入 `有限会社ODS`；采用蓝绿色表头、黑色表格边框、隔行浅色底纹；首页先给总体结论和关键风险；生成后逐页渲染检查。
