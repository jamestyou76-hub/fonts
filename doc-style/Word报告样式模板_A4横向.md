# Word 报告样式模板：A4 横向

适用场景：工程 TODO、设计需求清单、接口表、BOM/器件复核表等列数较多的技术文档。

## 1. 页面设定

| 项目 | 设定 |
|---|---|
| 纸张 | A4 |
| 方向 | 横向 Landscape |
| 页面尺寸 | 29.7 cm × 21.0 cm |
| 页边距 | 上 12 mm / 下 12 mm / 左 12 mm / 右 12 mm |
| 页眉页脚 | 页眉 8 mm / 页脚 8 mm |
| 推荐用途 | 5～7 列宽表格、工程 TODO、设计需求对照表 |

## 2. 字体与段落

| 项目 | 设定 |
|---|---|
| 主字体 | 等线 / DengXian |
| 英数字字体 | 等线 / DengXian |
| 日文文档 fallback | 等线优先；若日文假名显示异常，可用 Yu Gothic / Meiryo 作为 fallback |
| 正文字号 | 9.5 pt |
| 表格字号 | 8.5～9 pt |
| 标题 | 16 pt，加粗 |
| 一级标题 | 13 pt，加粗，蓝绿色或深灰色 |
| 二级标题 | 11 pt，加粗 |
| 段前/段后 | 标题段后 4～6 pt；正文段后 3 pt |
| 行距 | 单倍或固定值 12～13 pt |

## 3. 颜色规范

| 用途 | 颜色 | Hex |
|---|---|---|
| 表头底色 | 蓝绿色 | `#009999` |
| 表头文字 | 白色 | `#FFFFFF` |
| 奇数数据行 | 白色 | `#FFFFFF` |
| 偶数数据行 | 浅青绿色 | `#EAF7F7` |
| 表格边框 | 黑色 | `#000000` |
| 强调文字 | 深红/深橙，谨慎使用 | `#C00000` / `#C65911` |

## 4. 表格风格

| 项目 | 设定 |
|---|---|
| 表格边框 | 全部黑色实线 |
| 边框线宽 | 0.75 pt，必要时外框 1.0 pt |
| 表头 | 蓝绿色底色，白色文字，加粗，居中 |
| 数据行 | 隔行底色：白色 / 浅青绿色 |
| 单元格内边距 | 上下 1.5～2 mm，左右 1.5～2 mm |
| 对齐 | 编号/状态列居中；说明列左对齐；数值列居中或右对齐 |
| 换行 | 允许自动换行，避免缩小到不可读 |
| 表格宽度 | 100% 页面可用宽度 |
| 表格标题 | “表 1：xxx” 格式，置于表格上方或下方均可，全文保持一致 |

## 5. 横向版推荐表格列结构

### 工程 TODO / 设计需求表

| No. | 分类 | 当前 SCH 状态 | 设计需求 / TODO | 优先级 | 备注 / TBD |
|---:|---|---|---|---|---|
| 1 | 输入电源 | 已有 VBUS 输入 | VIN 输入能力需与充电电流设定匹配 | High | 5V/2A or 5V/3A = TBD |

### 器件/MLCC 复核表

| No. | 节点 | 当前器件 | 设计依据 | 设计判断 | 需求 / 修改方向 |
|---:|---|---|---|---|---|
| 1 | BAT | 10uF MLCC | BQ25887 推荐 BAT 电容 | EVM 同等配置 | 预留追加电容位 |

## 6. 内容写法规则

- 只写设计需求时，不写“测试”“验证”“实测”“后续确认”等执行阶段内容。
- 未确定项目统一写为 `TBD`。
- 状态表达建议使用：`OK` / `需修改` / `需确认` / `TBD`。
- 技术判断尽量短句化，避免长段落塞进表格。
- 关键限制条件写入备注列，例如：`VIN=5V/2A 时 1.5A 充电不可保证`。
- 对客户要求与当前 SCH 不一致处，优先写成“设计需求”，不要写成评价语。

## 7. Word 生成实现要点

使用 `python-docx` 生成 Word 时建议：

- `section.orientation = WD_ORIENT.LANDSCAPE`
- `section.page_width = Cm(29.7)`
- `section.page_height = Cm(21.0)`
- 所有 paragraph/run 设置 East Asian font 为 `等线`
- 表头填充 `009999`
- 表头文字设为白色、加粗
- 数据行隔行填充 `FFFFFF` / `EAF7F7`
- 所有单元格边框设置为黑色 `000000`
- 长表格允许跨页，并设置表头重复（如工具支持）

## 8. 文件命名建议

`项目名_文档主题_A4横向_语言_JST日期时间.docx`

生成文件名必须在扩展名前追加 JST 日期时间后缀，格式为 `_yyyy-mmdd-hhmm`，例如 `_1999-0322-1734`。

示例：

`电池充电基板_当前SCH电路TODO_精简设计需求_ZH_1999-0322-1734.docx`
